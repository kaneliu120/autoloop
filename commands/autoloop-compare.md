# AutoLoop T2: Compare — 多方案对比

## 执行前提

读取 `autoloop-plan.md` 获取：
- 要比较的选项列表（必须 ≥ 2 个）
- 评估维度列表（可自动生成）
- 权重配置（如果用户指定了优先级）

## 第一轮：并行选项分析

### 1.1 为每个选项分配独立 subagent

每个 analyzer subagent 的指令：

```
你是 option-analyzer subagent，负责深度分析以下选项：

选项名称：{选项 A}
对比主题：{主题}
评估维度：{维度列表}

任务：对这个选项在每个评估维度上进行深度分析。

要求：
1. 每个维度必须有具体的证据支撑（数据、引用、案例）
2. 识别核心优势（最多 3 个）
3. 识别核心劣势/风险（最多 3 个）
4. 识别最适合的使用场景
5. 识别不适合的使用场景

输出格式：
## 选项分析：{选项名称}

### 维度评分

| 维度 | 得分 (1-10) | 证据摘要 | 来源 |
|------|------------|---------|------|
| {维度 1} | {N} | {简短证据} | {URL} |

### 核心优势
1. {优势 1}：{具体说明，有数据支持}
2. {优势 2}：...
3. {优势 3}：...

### 核心劣势/风险
1. {劣势 1}：{具体说明，评估影响}
2. {劣势 2}：...

### 适用场景
✓ 适合：{场景描述}
✗ 不适合：{场景描述}

### 真实用户评价
（来自 Reddit/Stack Overflow/Hacker News 等）
- "{引用}"（来源：{URL}）

### 综合得分（加权前）
总分：{X}/10，置信度：{高/中/低}（基于 {N} 个信息来源）
```

### 1.2 独立的中立审查

所有选项分析完成后，运行一个 neutral-reviewer subagent：

```
你是 neutral-reviewer，负责检查以下分析是否存在偏见。

读取所有选项的分析结果，检查：
1. 评分是否有明显异常（某选项在所有维度都 ≥9 或 ≤3）
2. 证据质量是否均衡（某选项引用了更权威的来源）
3. 是否存在选择性引用（只引用对某选项有利的信息）
4. 评分标准是否一致（同等水平在不同选项中评分是否相同）

输出：
- 偏见风险评估（低/中/高）
- 需要重新评估的维度（如果有）
- 建议增补的对立证据（如果有）
```

## 第二轮：加权评分 + 敏感性分析

### 2.1 加权计算

```
综合得分 = Σ (维度得分 × 维度权重)
```

用户配置的权重优先，默认权重次之。

计算每个选项的加权得分，生成排名。

### 2.2 敏感性分析

测试推荐是否稳健：调整关键维度权重 ±10%，看推荐结果是否变化。

```
敏感性测试：

场景 1（功能匹配优先，+10% 权重）：
  排名：{选项} 第一，分差 {X} 分

场景 2（成本优先，+10% 权重）：
  排名：{选项} 第一，分差 {X} 分

场景 3（风险规避优先，+10% 权重）：
  排名：{选项} 第一，分差 {X} 分

结论：
- 推荐稳健性：{高（3/3 场景推荐同一选项）/ 中 / 低}
- 临界维度：{对结果影响最大的维度}
```

## 对比矩阵生成

生成完整的对比矩阵：

```markdown
## 综合对比矩阵

| 维度（权重） | {选项 A} | {选项 B} | {选项 C} |
|------------|---------|---------|---------|
| 功能匹配度 (25%) | 8/10 ✓ | 7/10 | 6/10 |
| 技术成熟度 (20%) | 9/10 ✓ | 8/10 | 7/10 |
| 学习曲线 (15%) | 5/10 | 8/10 ✓ | 7/10 |
| 社区生态 (15%) | 9/10 ✓ | 7/10 | 5/10 |
| 成本 (15%) | 4/10 | 7/10 ✓ | 9/10 |
| 长期风险 (10%) | 8/10 ✓ | 7/10 | 6/10 |
| **加权总分** | **7.5** | **7.3** | **6.7** |
| **排名** | **🥇 第 1** | **🥈 第 2** | **🥉 第 3** |

注：✓ 表示该维度最佳选项
```

## 推荐输出格式

```markdown
## 推荐结论

### 主推荐：{选项 A}

**推荐理由**：
{选项 A} 在核心维度（功能匹配度、技术成熟度、社区生态）均排名第一，
综合加权得分 7.5/10，领先第二名 0.2 分。

主要优势：
1. {优势 1}（数据支持：{来源}）
2. {优势 2}
3. {优势 3}

主要风险：
1. {风险 1}（缓解建议：{措施}）
2. {风险 2}

### 适用条件

✓ **选择 {选项 A} 当**：
- {条件 1}
- {条件 2}

✓ **选择 {选项 B} 当**：
- {条件 1}（尤其是 {场景}）

### 分差分析

选项 A vs B 差距仅 0.2 分，不稳健（成本权重提升 10% 后推荐反转）。
**建议**：如果成本约束严格，重新权衡 A vs B。

### 不推荐的理由

{选项 C}：{具体原因，不是笼统的"不如其他选项"}
```

## 质量门禁检查

在输出最终推荐前，验证：
- [ ] 每个选项在每个维度都有至少 1 个具体证据
- [ ] 关键维度有至少 2 个独立来源
- [ ] 偏见审查已通过（neutral-reviewer 结论）
- [ ] 敏感性分析已完成
- [ ] 置信度已明确声明
- [ ] 信息缺口已列出

未通过则进行第三轮补充研究。
