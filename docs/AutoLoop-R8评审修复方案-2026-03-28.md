## P0: 真实 Bug 修复（3 项）

### F1: 评分器与渲染器格式不兼容

**问题**: `autoloop-score.py` 期望 `## 维度` + 内联 `来源` 的 bullet 格式；`autoloop-render.py` 输出 `## 第N轮发现` + `### 维度 [confidence]` + 独立 `来源:` 行。评分器对渲染器产出会系统性低估。

**修复方案**: 统一为渲染器的格式（渲染器是 SSOT 的输出端，评分器适配渲染器）

**修改文件**: `scripts/autoloop-score.py`
- Section 分割：从 `re.split(r'\n##\s+', content)` 改为同时识别 `##` 和 `###` 层级
- 维度提取：从 `##` sections 改为从 `### 维度名 [confidence]` 提取
- Source 检测：从仅识别内联 `来源` 改为同时识别独立 `来源:` 行（下一行模式）
- Bullet 计数：从仅 `- ` 开头改为同时计数 `###` 下的段落内容

**验证**: 用 `autoloop-render.py` 生成一份 sample findings → 喂给修改后的 score.py → 确认评分非零

### F6: 阶段转换约束明确化

**问题**: 8 阶段序列在 loop-protocol.md:210-226 定义，但 state.py 的 update 命令无语义验证，也未引用阶段转换规则。

**修复方案**: 双层修复 — 协议层定义 + 脚本层引用

**A) 协议层**（`protocols/loop-protocol.md`）:
- 在阶段定义后（约 line 226）新增 `### 阶段转换约束` 子节：

```markdown
### 阶段转换约束

合法转换路径（严格顺序，不可跳跃、不可逆）：

| 当前阶段 | 合法下一阶段 | 约束 |
|---------|------------|------|
| OBSERVE | ORIENT | — |
| ORIENT | DECIDE | — |
| DECIDE | ACT | 决策锁定后不可回退 |
| ACT | VERIFY | — |
| VERIFY | SYNTHESIZE | — |
| SYNTHESIZE | EVOLVE | — |
| EVOLVE | REFLECT | — |
| REFLECT | OBSERVE (下一轮) / 已完成 | 终止需满足条件 |

**不可逆规则**: ACT 开始后，DECIDE 的选择不可修改。如需调整策略，必须完成当前轮次后在下一轮 DECIDE 中调整。
```

**B) 脚本层**（`scripts/autoloop-state.py`）:
- 添加 `PHASES` 常量和转换验证：

```python
PHASES = ["OBSERVE", "ORIENT", "DECIDE", "ACT", "VERIFY", "SYNTHESIZE", "EVOLVE", "REFLECT"]

def validate_phase_transition(current, target):
    """验证阶段转换合法性（参照 loop-protocol.md 阶段转换约束）"""
    if current not in PHASES or target not in PHASES:
        return False, "未知阶段: {} → {}".format(current, target)
    ci, ti = PHASES.index(current), PHASES.index(target)
    if ti == ci + 1:
        return True, ""
    if target == "OBSERVE" and current == "REFLECT":
        return True, "进入下一轮"
    return False, "非法转换: {} → {}（必须按顺序推进）".format(current, target)
```

- 在 `cmd_update` 中，当更新路径包含 `phase` 时调用验证

**验证**: 尝试从 OBSERVE 直接跳到 ACT → 确认报错
