# loop-data-schema ↔ autoloop-validate Mapping Table

Maintaining this table helps decide whether changing the schema requires changing validation logic. All implementation functions live in `scripts/autoloop-validate.py`.

| Schema / documentation section | Validation function | Notes |
|-------------------------------|---------------------|-------|
| Top-level `plan` / `iterations` / `findings` / `results_tsv` / `metadata` | `_check_top_level_structure` | Missing keys are errors |
| `iterations[].strategy_id` ↔ findings / strategy_history | `_check_primary_key_consistency` | Unregistered strategy is an error |
| `plan.dimensions` ↔ `iterations[].scores` keys | `_check_dimension_consistency` | Undefined dimension is an error; unused dimension is a warning (only for dimensions that appear in **non-exempt** `plan.gates` rows) |
| `plan.gates` contract (`dim` / `manifest_dimension` / non-manifest original name) | `_check_plan_gates_contract` | Promoted to an error in strict mode; **rows with `status=exempt/Exempt` are skipped entirely** (aligned with `autoloop_kpi.plan_gate_is_exempt`) |
| Final-round `phase` and the minimum DECIDE / VERIFY / REFLECT artifacts | `_check_phase_artifacts` | strict -> error; otherwise warning (including checkpoint and SSOT phase consistency) |
| `findings.rounds[].findings[]` summary / content | `_check_findings_canonical_fields` | Empty body is a warning; summary + content together is a warning |
| `results_tsv` columns and iteration continuity | `_check_tsv_completeness` | error |
| `iterations[].round` sequence | `_check_iteration_sequence` | error |
| `phase` enum and terminal state | `_check_phase_sequence` | unknown phase is an error; completion without REFLECT is a warning |
| `plan.gates[].current` ↔ latest scores | `_check_gate_status` | warning; exempt rows are skipped |
| `plan.budget` and round count | `_check_budget` | overspending is an error; mismatched `current_round` is a warning |
| `metadata.protocol_version` ↔ TSV rows | `_check_version_consistency` | error |
| `plan.decide_act_handoff.impacted_dimensions` ↔ final `results_tsv.side_effect` | `_check_side_effect_vs_handoff` | strict -> error; otherwise warning (P-03) |
| `iterations[-1].reflect` when final `phase=REFLECT` | `_check_phase_artifacts` | strict: requires `strategy_id` + `effect` (E-01); and must include one of `delta` / `rating_1_to_5` / legacy Likert `score` |

**Markdown fallback mode** (no `autoloop-state.json`) uses `validate_markdown` and is independent from the table above.

The official minimal SSOT example is `examples/minimal-state.json` at the repository root (generated by `autoloop-state.py init`; replace only `work_dir` / `task_id` to reuse it).
